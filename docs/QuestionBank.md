## 题库设计

1. 题库由很多很多个题目组成
2. 每个题目有这么几个字段：

- 题目 ID，数据库自动生成（无需手填）
- 排序序号（可选），用于控制题目在题库中的展示/抽题顺序
- 标题，例如：请你讲一下事件循环机制？
- 题目描述（可选），用于补充题目背景与上下文
- 参考答案（可选），AI 可以根据参考答案以及自己的知识水平，综合给面试者对某一道题的回答打分
- 要点（数组，可选），用于评估回答是否覆盖关键点
- 追问（数组，可选），用于补充追问线索
- 标签（数组，必填），用于方向分类与更精细的筛选与匹配（支持预置标签 + 自定义标签）
- 启用状态（是否上架）
- 难度等级，低、中、高
- 创建时间
- 更新时间

3. 后台题库管理

- 新建：当前支持点击新建题目来添加一道题目
- 删除：支持删除某一道题，也支持多选，同时删除多道题
- 编辑：支持对某一道题进行编辑，包括序号、标题、标签、参考答案、难度等级等
- 查询：用户可以根据字段信息来查询题目

- 后续扩展：可再支持上传 markdown 文件，由 AI 解析出多个题目后批量导入

4. AI 面试官使用题库

- 初期不做 RAG，优先通过数据库筛选 + 难度分层 + 随机抽题完成闭环
- 题量和复杂度上来后，再引入向量检索（RAG）

- AI 会对面试者的简历做判断，然后根据简历里的内容，拿出合适的题目对面试者进行提问

- 当题目数量达到一个数目或者面试时间到达一定时间的时候，停止本次面试，并且根据上下文对每个问题的回答进行打分（0-100分），最后加权平均计算（简单题错了扣分多，难题对了加分多），得出面试者最后这场面试的总分
